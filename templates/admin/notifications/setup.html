How It Works

Bulk sends are launched directly from the customer list. If rows are selected, only those customers are targeted. If nothing is selected, the current filtered result set becomes the target.

Operators choose a channel and a template in the customer-page modal. The system resolves the contact details for each customer, renders the template per customer, and queues notifications into the standard notification queue.

Email is the lowest-friction channel if SMTP sender profiles are already configured. SMS and WhatsApp depend on provider configuration before those sends will work in production.

Operator Checklist

  • 1. Confirm the channel shows as ready.
  • 2. Ensure at least one active template exists for that channel.
  • 3. Filter or select customers from the customer page.
  • 4. Use `Actions` → `Send message` and choose the template.
  • 5. Review queue and delivery history after the send is queued.
