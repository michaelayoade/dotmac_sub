{# Shared customer payment-intent submitter, used by pay.html and topup.html. POSTs the selected method to an intent endpoint and dispatches on the response: redirect_url -> bank-transfer instructions page charged -> saved card already charged server-side, go straight to verify checkout_url -> hosted gateway checkout (Flutterwave) flutterwave with no checkout_url -> throw (misconfigured) Returns {handled: true} when it navigated away, or {handled: false, intent} so the caller can open an inline gateway (Paystack) with the minted intent. Keeping this in one place stops pay.html and topup.html from drifting on the intent-response contract (idempotency header, redirect/charged/checkout branching). #}
