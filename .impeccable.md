# Design Context

## Users

**Primary: NOC Technicians** — Network operations staff monitoring and troubleshooting GPON/fiber connections daily. Context: often working on-call, under time pressure, with multiple tabs open (OLT CLI, Zabbix, Splynx, UISP). Job-to-be-done:

1. **Diagnose** — Why is an ONT offline? What's the optical signal? Where's the customer in the topology?
2. **Configure** — Push WiFi/PPPoE/firmware, change speed profile, authorize ONU.
3. **Monitor** — Glance at status across OLTs, spot PON outages, see authorization queues.

**Secondary: ISP admins, resellers, customers** — different portals (`/admin`, `/reseller`, `/portal`). These users are less technical; their screens can be less dense, more guided.

## Brand Personality

**Precise, confident, quiet.** The interface earns trust by being accurate and fast, not by being loud. Emotional goal: the operator feels *in control* — they see what's happening, they know the action they just took succeeded, they never wonder where a number came from.

## Aesthetic Direction

**Visual reference:** SmartOLT's information architecture (page patterns, density, tab groupings, KPI strips) rendered in the current dotmac Tailwind aesthetic. SmartOLT got the *what* right (density, every relevant metric one click away) but the *how* is tired Bootstrap-era chrome. Keep their pattern library, beat them on typography, spacing, dark mode, and interactive states.

**Anti-references:**
- Generic AI/SaaS gradients-and-purple look — too consumer, not serious.
- Stripe/Linear whitespace-heavy minimalism — eats screen real estate NOC operators need.
- Hardware-vendor UIs (Huawei iManager, Cisco DNA) — utilitarian but dated and unlovable.
- Decorative illustrations on data pages — reserve illustrations for empty states only.

**Tokens in play (already established, do not drift):**
- Fonts: **Outfit** (display/headings, `font-display`, tight tracking), **Plus Jakarta Sans** (body).
- Primary: teal/cyan `#06b6d4` (`primary-500`). Accent: warm orange `#f97316` (`accent-500`).
- Semantic: emerald=healthy, amber=warning, rose=critical, blue=info/network, slate=neutral.
- Surfaces: slate neutrals, full dark mode via `.dark` class.

## Design Principles

### 1. Status first, always
Online/offline/degraded state should be visible within 200ms of looking at any row. Use shape + color + position — never color alone. A red pill far from the context it describes is not status, it's noise.

### 2. Density with breathing room
NOC operators want more, not less, per screen. Default to compact rows (`py-2.5` not `py-4`), `text-sm` tables, grouped sections. But never crush — give each section a clear heading and a visible boundary (card border or divider).

### 3. Semantic color is a contract
Emerald = good, amber = needs attention, rose = broken. Never use these for decoration. If a button is rose, it's destructive or reporting a failure — full stop. Blue is reserved for network/primary action, violet for identity/people, slate for neutral metadata.

### 4. Actions confirm themselves
Every mutation (authorize ONU, push WiFi, reboot) must show inline success/failure without a full page reload. Spinner → toast or inline banner. HTMX swaps over page navigation whenever possible.

### 5. Dark mode is not an afterthought
Every new macro must pass light-mode and dark-mode review at author time, not later. No hardcoded hex; use the existing color utilities or CSS variables.

### 6. Typography does the hierarchy
`font-display` (Outfit) + tight tracking for titles. `tabular-nums` for anything numeric (signal, IPs, MACs, counts). Never bold a label and its value both — bold the value, dim the label.

### 7. Accessibility is WCAG AA
Text 4.5:1, UI 3:1. Every icon gets an `aria-label` or `sr-only` partner. Every form control has a `<label>`. Keyboard focus rings stay visible. Respect `prefers-reduced-motion` — animations are accents, never required signal carriers.

## Technical Constraints

- Tailwind CSS v4 with existing macro system (`templates/components/ui/macros.html`).
- Alpine.js for client-side state; HTMX for dynamic updates; Chart.js for charts.
- Jinja2 templates with macro components. New macros go in `templates/components/<group>/`.
- Light **and** dark mode mandatory.
- Dynamic Tailwind color classes used in macros MUST be safelisted in `tailwind.config.js`.
