# Design Context

## Target Audience
**NOC Technicians** — Network operations center staff who monitor and troubleshoot fiber connections daily. They need to quickly identify issues, understand device status at a glance, and take action efficiently.

## Use Cases
1. **Diagnose issues** — Quickly identify why an ONT is offline or performing poorly
2. **Configure devices** — Push WiFi settings, PPPoE credentials, firmware updates
3. **Monitor status** — Check signal levels, connection status, subscriber info at a glance

## Brand Personality
**Clean Professional** — Polished corporate feel with clear visual hierarchy. The interface should feel trustworthy, efficient, and precise without being cold or sterile.

## Design Principles

### Visual Hierarchy
- Status indicators (online/offline) must be immediately visible
- Signal quality should use semantic color coding (green/amber/red)
- Critical metrics (OLT Rx, ONU Rx) should be prominent
- Secondary information should be discoverable but not overwhelming

### Color Language
- **Emerald/Teal**: Online, healthy, good signal
- **Amber/Orange**: Warnings, degraded signal, attention needed
- **Rose/Red**: Offline, critical issues, errors
- **Blue/Indigo**: Network, topology, technical information
- **Slate**: Neutral, secondary information

### Information Density
NOC technicians prefer higher information density. Avoid excessive whitespace but maintain clear groupings. Use compact layouts with clear labels.

### Interaction Patterns
- Immediate visual feedback on actions
- Status changes should be obvious without page refresh
- Configuration changes should confirm success/failure clearly
- Use inline actions where possible, modals sparingly

## Technical Constraints
- Tailwind CSS v4 with existing macro system
- Alpine.js for client-side state
- HTMX for dynamic updates
- Jinja2 templates with macro components
- Must support light and dark modes
